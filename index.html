This extension bundles together several Lua filters üåô that will work with any Pandoc version above 2.17.

This extension allows for:

  • The creation of multiple bibliographies (or bibliographies with multiple sections, such as primary and secondary sources).
  • The citation of arbitrary fields1 of the references, such as author, editor, and translator names, or title and original-title of works.
  • The creation of linked indexes of citations.

Justification ü§î

BibLaTeX users trying out Citeproc would probably be exceptionally pleased with its speed and reliability, and the fact that one can convert bibliography consistently into all formats supported by Pandoc (including HTML, LaTeX, Word, and dozens of others).

Some limitations, however, would inevitably become apparent. First, there is the inability to split the bibliography into multiple sections. Then, there is the lack of commands familiar to BibTeX users, such as \citetitle, \citeauthor, \citefield. Finally, there is the lack of the interesting backref option afforded by bibtex and hyperref, which allows for the creation of linked indexes of citations.

This extension aims to provide a solution to these limitations and, hopefully, allow Pandoc and Quarto users to completely replace BibTeX/BibLaTeX with Citeproc.

Installation üì¶

Install using:

quarto install extension bcdavasconcelos/citetools

Activate it by adding citetools to the _quarto.yml file in the filters section.

filters:
  - citetools

If you need a sample project to test it out, you can download the citetools repository

Quickstart üöÄ

Add the path to the bibliography files to the document YAML header or to _quarto.yml üìÅ

---
bibliography_bib-name: path/to/bibfile.bib
bibliography_other-bib-name: path/to/otherbibfile.bib
---

Place the corresponding placeholders where they should be printed in the document body üñ®Ô∏è

::: {#refs_bib-name}
:::

::: {#refs_other-bib-name}

Cite references as you normally do in Pandoc and they will be linked üîó and backlinked üîó to the bibliography üìö

[@citekey]

Use [@Citekey]{.csl_field} to arbitrarily print specific fields instead of the usual Author-Date üí™

[@citekey]{.title}
[@citekey]{.original-title}

Append the field name with a dot (.) to avoid creating a link to the bibliography in this particular citationüìç

This can also be done globally by setting link-fields: false in the metadata2 üåê

---
